# STEPWORK 产品需求文档（PRD）

**版本：V1.0**  
**状态：立项基线**  
**上位文档：PRODUCT_CHARTER.md V2.0**  
**适用版本：Phase 0—V1.0**  
**产品形态：本地优先、Agent-Native 的开源 AI 内容运营桌面工作台**

---

## 1. 文档目的

本 PRD 定义 STEPWORK 要解决的用户问题、目标用户、核心流程、功能需求、优先级、交互约束和验收标准。

本文件不规定具体代码结构、协议字段和数据库表；相关内容由 `SYSTEM_SPEC.md` 约束。

---

## 2. 产品背景

创作者的内容生产通常分散在浏览器、聊天式 AI、文档、素材目录、剪辑软件、平台后台和多个 Agent 之间。现有工具普遍存在以下问题：

1. 参考内容分析、脚本创作、素材处理和发布彼此割裂；
2. AI 生成结果停留在聊天记录，无法沉淀为可管理内容资产；
3. 创作者已有的 Codex、Hermes、OpenClaw 或定制 Agent 无法安全进入内容工作流；
4. 云端产品要求上传素材、API Key 或账号会话，用户缺少数据控制权；
5. 多平台发布工具与内容创作脱节，自动化过程缺乏预览、授权和审计；
6. 同一账号的品牌声音、历史选题和用户修改偏好无法持续积累。

STEPWORK 通过本地桌面工作台、统一内容模型、Agent 互操作层和插件生态，将上述环节连接为可追溯工作流。

## 3. 产品目标

### 3.1 核心目标

1. 用户可以在本地完成“导入素材—分析—原创角度—脚本—视频草稿—平台适配”的闭环；
2. 所有重要结果均沉淀为结构化 Artifact，而不是只存在于聊天记录；
3. 用户可以通过桌面 UI、CLI 和 MCP 使用同一套核心能力；
4. 后续可以通过 A2A 与 ACP 接入外部 Agent，而无需重写核心业务；
5. 发布辅助默认保留人工最终确认；
6. 模型、平台、语音、渲染和 Agent 均可通过插件替换；
7. 用户掌握项目数据、凭据和浏览器会话。

### 3.2 用户价值目标

- 在 15 分钟内从参考素材得到一个可编辑的原创脚本；
- 在 30 分钟内从脚本得到一个可继续编辑的视频草稿；
- 用户能清楚看到内容来源、AI/Agent 参与过程和最终发布版本；
- 用户已有 Agent 能读取受控项目上下文并向 STEPWORK 提交结果；
- 用户在不购买官方云服务时仍可完成核心流程。

### 3.3 非目标

V1.0 之前不以以下能力为核心目标：

- 专业非线性剪辑；
- 百账号无人值守矩阵；
- 去水印、搬运或规避版权识别；
- 设备指纹伪装和风控规避；
- 自动预测爆款；
- 取代用户已有的通用 Agent；
- 构建封闭的云端内容 SaaS。

## 4. 目标用户

### 4.1 个人创作者

**特征**

- 每周生产 2—10 条内容；
- 已使用一种或多种 AI/Agent 工具；
- 经常参考同行视频、文章或评论；
- 需要保留个人表达风格；
- 不愿将全部素材和账号数据上传云端。

**核心任务**

- 快速理解参考内容；
- 找到适合自己账号的原创角度；
- 生成并修改脚本；
- 输出字幕、配音和视频草稿；
- 将结果交给其他 Agent 或剪辑工具继续处理。

### 4.2 1—5 人内容团队

**核心任务**

- 管理多个内容项目和账号；
- 共享品牌资料、脚本和素材；
- 进行版本比较和审核；
- 让不同 Agent 或成员承担研究、创作、核查和发布任务；
- 保留完整操作记录。

### 4.3 企业内容与电商团队

**核心任务**

- 接入商品、品牌和企业知识库；
- 使用企业内部模型或 Agent；
- 执行审批、敏感词和合规流程；
- 私有部署；
- 接入 CMS、PIM、飞书、企业微信等内部系统。

### 4.4 开发者与插件作者

**核心任务**

- 开发 AI、Source、Speech、Renderer、Publisher 或 Agent Adapter 插件；
- 通过 CLI/MCP 调用 STEPWORK；
- 使用标准 Schema 交换 Artifact；
- 测试平台适配器和选择器。

## 5. 核心用户任务（JTBD）

1. 当我看到值得参考的内容时，我希望快速知道它使用了什么结构，以便决定是否值得借鉴；
2. 当我需要创作新内容时，我希望结合自己的账号定位得到多个原创角度，而不是简单仿写；
3. 当 AI 或外部 Agent 给出结果时，我希望它进入项目版本体系，而不是消失在聊天记录；
4. 当我要生成视频时，我希望先得到一个可靠草稿，再进入专业剪辑；
5. 当我要发布内容时，我希望系统替我完成重复填写，但最终账号和内容必须由我确认；
6. 当我已有常用 Agent 时，我希望它能安全读取项目、调用工具和提交结果；
7. 当我更换模型、平台或工具时，我希望历史项目和工作流不需要重建。

## 6. 核心用户流程

### 6.1 参考内容转原创脚本

1. 用户创建项目；
2. 导入本地视频、音频、逐字稿或链接；
3. 选择快速分析或精确分析；
4. 系统输出结构化分析报告；
5. 系统结合 BrandProfile 生成 3—5 个原创角度；
6. 用户选择角度并生成脚本；
7. 用户在编辑器中修改；
8. 系统保存 ContentVersion 和 Provenance；
9. 用户导出或继续生成视频草稿。

### 6.2 外部 Agent 参与项目

1. 用户在 Agent Connections 中连接 Agent；
2. 用户授予指定 Workspace/Project 权限；
3. Agent 通过 CLI、MCP、A2A 或 ACP 获取受控上下文；
4. Agent 发起分析、研究或编辑任务；
5. STEPWORK 展示任务进度和权限请求；
6. Agent 输出转化为 AgentArtifact；
7. 用户接受、拒绝或合并 Artifact；
8. 接受后的结果进入正式内容版本。

### 6.3 视频草稿生成

1. 用户选择 ContentVersion；
2. 选择语音 Provider、画面比例和 Renderer 模板；
3. 系统显示预计耗时、模型和费用；
4. 创建 RenderJob；
5. 用户可查看进度、取消或重试；
6. 输出 MP4、字幕、音频、封面和渲染记录；
7. 用户预览并选择导出或创建平台版本。

### 6.4 发布辅助

1. 用户选择 PlatformVariant 和 SocialAccount；
2. Publisher 插件执行内容校验；
3. 系统打开本地浏览器 Profile；
4. 插件上传媒体并填写内容；
5. STEPWORK 展示最终预览、账号、插件版本和 AI 标识；
6. 用户确认一次性 PublishApproval；
7. 插件执行最终发布；
8. 系统验证结果并保存证据。

## 7. 信息架构

桌面端一级导航：

1. **Home**：最近项目、待审批任务、失败任务和 Agent 状态；
2. **Projects**：项目、素材、分析、脚本、平台版本和 Artifact；
3. **Create**：快速导入、分析、脚本和视频草稿；
4. **Agents**：连接、会话、任务、Artifact Inbox；
5. **Publish**：账号、准备任务、发布审批和历史；
6. **Plugins**：插件目录、权限、状态和更新；
7. **Settings**：模型、语音、存储、安全、CLI/MCP/A2A/ACP。

项目内部导航：

- Overview；
- Sources；
- Analysis；
- Topics；
- Script；
- Assets；
- Render；
- Platforms；
- Agent Activity；
- Provenance。

## 9. Agent 与发布审批规则

### 9.1 默认禁止外部 Agent 直接执行

以下操作默认只能创建准备任务：

- 发布；
- 删除项目或资产；
- 覆盖原文件；
- 向外部服务上传原素材；
- 安装插件；
- 读取平台凭据；
- 使用高费用模型。

### 9.2 审批展示内容

审批界面至少展示：

- 请求者；
- 协议或插件；
- 目标 Workspace/Project；
- 将读取、修改或上传的数据；
- 费用或风险；
- 有效期；
- 一次性或持久授权范围。

## 10. 关键 UX 原则

1. **项目优先于聊天**：聊天是交互方式，项目和 Artifact 才是资产；
2. **进度可见**：长任务必须有阶段、进度、取消和失败原因；
3. **失败可恢复**：登录失效、模型失败、渲染失败和 Agent 超时均提供下一步；
4. **来源可见**：分析和外部 Agent 结果显示来源及信任等级；
5. **高风险操作减速**：发布、删除、上传和权限升级必须增加明确确认；
6. **默认不过度自动化**：首版优先建议、填写和预览；
7. **高级能力渐进展示**：普通创作者不必理解所有协议和插件概念。

## 11. 非功能需求

### 11.1 性能

- 桌面应用冷启动目标：P50 小于 5 秒；
- 本地项目列表打开：P50 小于 1 秒；
- UI 操作不得被视频任务阻塞；
- 长任务必须异步执行；
- 任务状态更新延迟目标小于 1 秒。

### 11.2 稳定性

- 应用异常退出后，已持久化任务可恢复；
- 不允许重复发布事故；
- 项目版本保存必须原子化；
- 数据迁移失败时保留旧数据和备份。

### 11.3 安全

- 密钥不明文存储；
- 本地协议端点默认只监听 Loopback；
- 插件和 Agent 按 Scope 授权；
- 高风险操作可审计；
- 日志不包含 Cookie、Token、二维码和验证码。

### 11.4 可扩展性

- Core 不硬编码模型、平台和 Agent 品牌；
- 插件接口和 Artifact Schema 版本化；
- UI、CLI 和协议入口共用 Application Services。

### 11.5 可访问性

- 主要操作支持键盘；
- 关键状态不只依赖颜色；
- 错误信息提供明确文本；
- 长任务状态支持屏幕阅读器可识别文本。

## 13. MVP 验收指标

### 13.1 产品验收

- 10 名种子用户中至少 7 名完成一次完整流程；
- 首次价值时间中位数不超过 20 分钟；
- 分析报告转脚本率不低于 60%；
- 脚本保存率不低于 50%；
- 视频草稿导出率不低于 30%；
- 7 日内至少 4 名用户再次创建项目。

### 13.2 工程验收

- 本地任务成功率不低于 90%；
- 应用重启后任务状态恢复率 100%；
- 项目数据丢失事故为 0；
- 重复任务导致重复输出可被幂等机制阻止；
- CLI 与 UI 对同一命令产生一致业务结果；
- MCP 高风险工具不得绕过审批策略。
