## 3. 各 W 范围与 Gate（来自 docs/MVP_PLAN.md）
### W7 — CLI + MCP Server
- **目标**：命令行入口 + MCP server，均调用同一 Command Bus。
- **范围**：
  - CLI：参数解析 → `buildEnvelope` → `dispatchCommand` → handler(`handle(env, deps)`)
  - MCP：将核心命令暴露为 tools（带 schema），权限边界与 UI 一致
- **Gate**：CLI / UI / MCP 对同一命令产出一致结果；MCP 不可越权（actor 白名单 `user`/`desktop` + 工具权限边界）
- **关键文件 / 新增**：`worker/runtime/commands/bus.py`、`handlers/commands.py`、`dev_bridge.py`；新增 `cli/`、`mcp/`
- **安全关注（security-officer）**：MCP 越权面、SSRF（已前置 resolve 守卫）、密钥不出内存覆盖层

### W8 — 插件 / BrandProfile / Provenance
- **目标**：插件加载机制 + 品牌档案 + 来源追踪。
- **范围**：
  - 插件沙箱 + 禁用开关（Gate：禁用后系统稳定；无插件时核心功能可用）
  - BrandProfile 读写 settings brand 段
  - Provenance 元数据（产出物溯源）
- **依赖**：settings brand 段（已落地）
- **Gate**：禁用任意插件后系统仍稳定；无插件时核心功能可用

### W9 — 集成 / 数据迁移 / 种子测试 → RC1
- **目标**：端到端集成 + 旧数据迁移 + 种子数据/测试 → RC1。
- **范围**：集成测试套件；数据迁移脚本；种子测试数据
- **Gate**：无数据丢失、无高危安全问题、P0 完成率 ≥ 80%
