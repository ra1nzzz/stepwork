## 2. 架构与决策

### 2.1 核心决策
1. **BrandProfile 不独立成表**：MVP_PLAN"基础字段"= 当前 5 字段（name/audience/tone/mustExecute/defaultOutput），继续走 `settings.brand` 段。多 profile 切换推 V0.2。`brand_profile_id` 列保留但 W8 不用。
2. **Provenance 只做只读聚合页**：W8 不改 3 个 AI handler 的写入路径（避免回归风险）。`ProvenanceView` 从 `content_versions.producer` + `provenance_records` 表做 best-effort 聚合。统一写入路径推 W9。
3. **插件不真起子进程**：W8 MVP 只做 Manifest 解析 + 启停状态管理 + 权限展示。ADR-009 独立进程运行时属 V0.2。示例插件以"已注册未启用"形态存在，演示 Gate"禁用后稳定"。
4. **Agent 占位**：表已建但 W8 只做只读列表查询 + UI 空态。真实 A2A/ACP 属 V0.3。
5. **日志文件落盘**：新增 `$STEPWORK_HOME/logs/worker.log`（按 `retentionDays` 轮转），为诊断包提供素材。

### 2.2 共享接缝（Layer 0，必须最先）
- `schemas/command-envelope.schema.json` commandType enum 扩展（+9 命令，13→22）
- `worker/runtime/commands/bus.py` `_ROUTES` 路由注册
- 新增 commandType：`ListPlugins` / `GetPluginManifest` / `EnablePlugin` / `DisablePlugin` / `GetProvenance` / `ListAgentTasks` / `ListAgentArtifacts` / `GetAgentTask` / `ExportDiagnosticsBundle`

## 4. 决策

| # | 决策 | 理由 |
|---|---|---|
| D1 | BrandProfile 走 settings.brand，不建表 | MVP"基础字段"= 5 字段已满足；多 profile 切换无真实场景驱动 |
| D2 | Provenance 只读聚合，不改 AI handler 写入路径 | 避免 W8 回归 3 个已稳定 handler；统一写入推 W9 |
| D3 | 插件只做 Manifest + 启停状态，不起子进程 | ADR-009 独立进程是 V0.2；W8 Gate"禁用后稳定"用状态管理即可演示 |
| D4 | Agent 只做只读列表 + UI 空态 | 表已建但无真实 Agent 接入；V0.3 才做 A2A/ACP |
| D5 | 日志加文件 handler（`$STEPWORK_HOME/logs/worker.log`） | 诊断包需要日志素材；当前只走 stderr 无法收集 |
| D6 | 诊断包按 `data.desensitize` 脱敏 | 复用 `config.py:_mask_secrets`（L95-111）已验证的脱敏逻辑 |
| D7 | Sidebar 新增 4 项（plugins/provenance/agent/diagnostics） | 顶层可见性 > 收进 settings；MVP 演示价值更高 |
| D8 | commandType enum 一次性 +9（Layer 0 单独 PR） | 避免 5 条支线并行修改同一 schema 文件冲突 |

## 5. 风险与应对

| # | 风险 | 级别 | 应对 |
|---|---|---|---|
| R1 | `provenance_records` 表无 Python 层，ProvenanceView 聚合时 `producer` 形状不统一（analyze/topic/script 三种） | P1 | best-effort 聚合 + 空态兜底；统一形状推 W9 |
| R2 | 0003 placeholder 表列名与 SYSTEM_SPEC §8 描述风格不一致 | P2 | 以 SQL 列名为准，spec 文字作语义参考 |
| R3 | 日志文件落盘可能触碰磁盘权限 | P1 | 路径走 `$STEPWORK_HOME`（已有），失败时降级到 stderr 不阻塞 |
| R4 | 诊断包含敏感信息（密钥/路径） | P0 | 默认 `desensitize=True`，复用 `_mask_secrets`；测试断言无明文密钥 |
| R5 | 插件 Manifest 解析失败导致 worker 启动崩溃 | P0 | loader 全程 try/except，单个插件解析失败标记 `status=error` 不影响其他 |
| R6 | Sidebar 新增 4 项后导航过长 | P2 | MVP 可接受；后续可分组或收进 settings |
| R7 | Agent 占位页面空态用户体验差 | P2 | 空态文案明确"V0.2 启用"，不放虚假数据 |
