# W7_PLAN.md — CLI + MCP Server（共享 Command Bus）

**YTDEV 阶段**：Phase 1 调研完成（排障手审计 + 产品评审员 gap 分析），本文件为 Phase 2 最终 PLAN。
**Gate（来自 MVP_PLAN）**：CLI / UI / MCP 对同一命令产出一致结果；MCP 不可越权。
**依赖**：无前置（设置页已落地 GetConfig/UpdateConfig + ConfigSpec，是地基）。后续 W8/W9 依赖本 W 的总线接入点。

## 1. 调研核心结论（已采信两位成员）
- **复用可行、零新增命令逻辑**：`worker/runtime/commands/bus.py:dispatch` 是纯 async 函数；dev_bridge 已证明可进程内 `asyncio.run(commands.handle_command(...))` 同步走。CLI/MCP 无需起 HTTP server。
- **权限边界由总线担保**：config 命令在 bus 强制 `actor∈{user,desktop}`（`_ALLOWED_CONFIG_ACTORS`）；其余命令无 actor 限制。MCP 若自报 `actor.type∈{agent,plugin,system}` → 配置读写被自动拒绝 = "不可越权"由总线担保。但总线**信任调用方自报 actor**（白名单是约定非硬边界），故**权威边界 = MCP server 注册的 tool 集合**，而非 actor.type。
- **密钥安全现状**：`GetConfig` 输出已掩码（`••••`），`resolved` 仅返 `hasKey:bool`；`CONFIG_OVERRIDES` 是内存层（dev_bridge 单进程跨请求存活，一次性 CLI 跑完即失 → CLI 调 AnalyzeSource 只能回退 env）。

## 2. 架构与最小接缝
新增**唯一共享入口** `worker/runtime/app.py`：
```python
async def run_command(raw: dict, *, db_path=None) -> dict:
    # bootstrap WorkerState + bootstrap_db
    # 构造 Deps(repos=Repos(...), ingest, asr, ai, tts, renderer)
    # env = parse_envelope(raw); return (await dispatch(env, deps)).model_dump()
```
CLI 与 MCP **都只 import `run_command`**，所有命令逻辑留在 `handlers/`，零新增命令代码。
同时在 `app.py` 提供共享 `build_envelope(*, command_type, source, actor_type, workspace_id="ws-local", **payload)` —— 把 TS 里硬编码的 `buildEnvelope`（`apps/desktop/src/lib/tauri.ts`）抽成参数化 Python 版，统一 `source∈{ui,cli,mcp}`、`actor.type`、`workspace_id` 默认 `ws-local`，否则三端结果可能不一致（直接击穿 Gate）。

|---|---|---|
| `config get` | GetConfig | |
| `config set --file <json>` | UpdateConfig | 密钥**只走 --file/stdin/env，绝不走命令行参数**；输出永不回显明文 |
| `analyze` | AnalyzeSource | |
| `topic generate` | GenerateTopic | |

| `script save` | SaveScript | |

### MCP server（新增 `mcp/server.py`，常驻进程，pyproject 加 `stepwork-mcp`）
**推荐「只读」路线**——最小 tool 清单（与 ADR-007 的 5-tool 意图对齐）：
1. `get_config` → GetConfig（返回掩码 detail 原样）
2. `list_projects` → **新增** ListProjects（只读）
3. `get_project` → **新增** GetProject（只读）
4. `get_job_status` → **新增** GetJobStatus（只读）
5. `analyze_source` → AnalyzeSource
- **永不注册 `UpdateConfig`**（写密钥），从根上保证不越权。
- 每个 tool 构造 envelope（`source="mcp"`, `actor.type="agent"`）→ `run_command` → 返掩码 detail。
- 禁止任何读取 `CONFIG_OVERRIDES`/DB 原始值的自定义 tool。

## 4. 待拍板的决策（lead 推荐默认，可推翻）
1. **MCP 只读路线需新增 3 个只读 query 命令**（ListProjects/GetProject/GetJobStatus，trivial，仅读 repos）。→ 推荐**在 W7 内新增**（最干净地满足"MCP 不越权"Gate）。若不想加命令，降级为基于现有命令的 `get_config`+`generate_topic`+`generate_script`+`save_script`+`analyze_source`（低风险但非纯只读，会写 content 版本）。
2. **CLI stretch 命令**（import/transcribe/render/cancel）W7 是否一并做？→ 推荐**延后**（先交付核心 5 条 + MCP 只读）。
3. **workspaceId 默认统一为 `ws-local`** → 必需（一致性地基），直接采用。

## 8. 接口签名（核心契约）

```python
# L.20 worker/runtime/app.py
async def run_command(raw: dict, *, db_path: str | None = None) -> dict:
    """进程内执行一条命令；返回 CommandResult.model_dump()。
    任何异常包裹为 {"ok": False, "error": str(e)}，调用方永不崩溃。"""

def build_envelope(*, command_type: str, source: str, actor_type: str,
                   workspace_id: str = "ws-local",
                   project_id: str | None = None,
                   idempotency_key: str | None = None,
                   payload: dict | None = None,
                   command_id: str | None = None,
                   request_id: str | None = None) -> dict:
    """构造 command-envelope.schema.json 信封。复刻 apps/desktop/src/lib/tauri.ts
    的 buildEnvelope 形状，参数化 source/actor/workspace_id —— 三端一致性根基。"""

# L.21 worker/runtime/handlers/queries.py
async def handle(env: CommandEnvelope, deps: Deps) -> CommandResult:
    """路由 ListProjects / GetProject / GetJobStatus，全部只读 SELECT，不写库。
    - ListProjects：按 workspaceId 列 content_projects（DESC created_at）
    - GetProject：兼容 payload.projectId / payload.project_id / env.projectId
    - GetJobStatus：兼容 payload.jobId / payload.job_id；走 deps.repos.jobs.get
    未知 commandType → DispatchError("UNKNOWN_COMMAND")。"""

# L.23 worker/runtime/commands/bus.py
_ROUTES: dict[str, str] = {
    ...,
    "GetConfig": "worker.runtime.handlers.config",
    "UpdateConfig": "worker.runtime.handlers.config",
    "ListProjects": "worker.runtime.handlers.queries",
    "GetProject": "worker.runtime.handlers.queries",
    "GetJobStatus": "worker.runtime.handlers.queries",
}
_ALLOWED_CONFIG_ACTORS: tuple[str, ...] = ("user", "desktop")
# 仅 UpdateConfig 受 actor 白名单限制；GetConfig 返掩码，对所有合法 actor 开放。

# L.24 cli/__main__.py
SOURCE = "cli"; ACTOR_TYPE = "desktop"; DEFAULT_WORKSPACE_ID = "ws-local"
def main(argv: list[str] | None = None) -> int  # 0 成功 / 2 参数错 / 1 dispatch 错
# 子命令：config {get,set --file|--stdin} / analyze / topic generate / script {generate,save}
# 密钥安全：config set 互斥必填 --file/--stdin，绝不接收 --api-key 等明文参数。

# L.25 mcp/server.py
_TOOL_COMMANDS: dict[str, str] = {
    "get_config": "GetConfig",
    "list_projects": "ListProjects",
    "get_project": "GetProject",
    "get_job_status": "GetJobStatus",
    "analyze_source": "AnalyzeSource",
}  # 永不出现 "update_config" / "UpdateConfig" —— 根授权保证
TOOLS: list[dict] = [...]  # 固定 5-tool catalogue，inputSchema 手写
async def _call_tool(name, arguments) -> dict  # build_envelope(source="mcp", actor_type="agent") → run_command
def main() -> None  # stdio JSON-RPC 读循环（line-delimited JSON）
```

## 9. 三角色头脑风暴补丁（P0/P1）
延续 §5 的 R1–R4，按角色视角再拆一遍，确保不漏：

| 角色 | 发现 | 级别 | 合入 |
|---|---|---|---|
| 架构师 | `run_command` 必须**进程内** `asyncio.run`，**绝不 spawn worker 子进程**（沙箱无 cargo/DISPLAY，子进程起不来；dev_bridge 已证同进程可行） | P0 | L.20 |
| 架构师 | `build_envelope` 是三端一致性根基，必须参数化 `source`/`actor_type`/`workspace_id`；TS 的 `buildEnvelope` 硬编码与 Python 版必须形状一致（否则击穿 Gate） | P0 | L.20 |
| 安全健壮性 | **MCP 权威边界 = tool 注册集**（非 actor.type 自报）：`update_config` 永不注册 = 写密钥路径根本不可达。actor 白名单是约定层二道防线 | P0 | L.25 + L.23 |
| 安全健壮性 | `GetConfig` 读开放给 agent（返掩码 `••••` + `hasKey:bool`），否则 MCP `get_config` 真实路径被击穿。原总线把 GetConfig 与 UpdateConfig 一并限白名单 = bug | P0 | L.23（修 bus） |
| 安全健壮性 | CLI `config set` 互斥必填 `--file`/`--stdin`，**绝不**暴露 `--api-key`/`--secret`/`--token`/`--password` 参数；输出永不回显明文密钥 | P0 | L.24 |
| QA | Gate 测试 = 同命令经 CLI / UI(mock) / MCP 产出**相同 CommandResult**（commandType/source/actor 之外的字段一致） | P0 | L.26 |
| QA | MCP `tools/list` 必须返回**恰好 5** tool 且不含 `update_config`；`_TOOL_COMMANDS` 反查也不含 `UpdateConfig`（defense in depth） | P0 | L.26 |
| 产品 | MCP 只读 5-tool 与 ADR-007 的 5-tool 意图对齐；新增 3 个只读 query 命令（ListProjects/GetProject/GetJobStatus）是"最干净满足 MCP 不越权 Gate"的路径 | P1 | L.21/L.22/L.23 |
| 产品 | CLI stretch（import/transcribe/render/cancel）延后 —— W7 先交付核心 5 条 + MCP 只读 | P1 | §13 范围外 |
| 健壮性 | `run_command` 顶层 `except Exception` 兜底为 `{"ok": False, "error": str(e)}`，避免击垮 CLI/MCP 调用方 | P1 | L.20 |
| 健壮性 | CLI 一次性进程丢失 `CONFIG_OVERRIDES` 内存密钥 → 文档明示 CLI 依赖 env / 持久化（掩码）settings；MCP 常驻无碍 | P2 | §13 文档 |
