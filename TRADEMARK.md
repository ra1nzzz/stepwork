# STEPWORK 商标政策

**生效日期**：2026-07-21
**适用范围**：STEPWORK 名称、Logo、相关标识

## 保留的权利

"STEPWORK"、STEPWORK Logo 及相关视觉标识（统称 "STEPWORK 商标"）由项目维护者保留所有权利。

**重要**：本项目的代码许可证（AGPL-3.0 / Apache-2.0）**不**延伸授权您使用 STEPWORK 商标。商标与版权是独立的法律概念。

## 允许的使用（无需事先许可）

### 1. 描述性使用（Nominative Fair Use）

您可以：

- 在文字中真实地引用 "STEPWORK"，例如"我使用 STEPWORK 来管理内容工作流"
- 撰写关于 STEPWORK 的文章、教程、评测、对比
- 在学术研究中提及 STEPWORK

**要求**：必须真实、不引起混淆、不暗示官方背书。

### 2. 兼容性声明

您可以：

- 声明您的产品"兼容 STEPWORK"或"基于 STEPWORK SDK 开发"
- 使用 "for STEPWORK"、"STEPWORK-compatible" 等说明性短语

**要求**：您的产品名称不能与 STEPWORK 混淆。例如：

- ✅ "Acme Plugin for STEPWORK"
- ✅ "STEPWORK Exporter by Acme"
- ❌ "Acme STEPWORK"
- ❌ "STEPWORK Pro"

### 3. Fork 注明

您可以 fork 本项目并继续开发，但必须：

- **重命名您的 fork**：不能使用 "STEPWORK" 作为产品名或域名主体
- **注明来源**：在 README 与关于页面显著位置注明 "Forked from STEPWORK (https://github.com/ra1nzzz/stepwork)"
- **修改 Logo**：不得使用原 Logo 或其变体

**示例**：

- ✅ "MyWork — Forked from STEPWORK"
- ❌ "STEPWORK Community Edition"
- ❌ "STEPWORK-Plus"

## 需要事先许可的使用

以下使用必须获得书面许可（联系 hello@stepwork.dev，占位）：

- 将 "STEPWORK" 用作您产品/公司名称的一部分
- 使用 STEPWORK Logo 或其修改版
- 注册包含 "stepwork" 的域名（除 `*.stepwork.dev` 二级子域用于官方镜像）
- 在商业会议、广告中以 STEPWORK 名义进行推广
- 暗示官方背书、赞助或合作

## 品牌资源使用规范

获得许可后使用 STEPWORK Logo 时：

- 不得改变 Logo 的比例、颜色、字体
- 不得将 Logo 与其他图形混合
- 周围留白 ≥ Logo 高度的 1/2
- 不得用作您自己产品的主要标识

## 域名与社交账号

- `stepwork.dev` 及其子域为官方保留（占位，待注册）
- GitHub organization `stepwork`、Twitter/X `@stepwork` 等社交账号为官方保留

## 违规处理

如发现商标侵权，维护者将：

1. 首先友好沟通要求整改（30 天整改期）
2. 沟通无果后，向托管平台（GitHub、npm、PyPI 等）发起商标投诉
3. 严重或恶意侵权将寻求法律途径

## 合理使用免责

本政策不影响您在适用法律下享有的合理使用（fair use）权利。

本政策可能修订。重大变更将提前 30 天在 GitHub Discussions 公告。
