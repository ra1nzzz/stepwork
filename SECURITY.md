# Security Policy

## 支持的版本

| 版本 | 支持状态 |
|---|---|
| `main` 分支 | ✅ 持续接收安全修复 |
| 最新 Release | ✅ 接收安全修复直至下一版本发布 |
| 历史版本 | ❌ 不再维护，请升级 |

## 报告漏洞

**请勿在公开 Issue、Discussion 或 Pull Request 中披露安全漏洞。**

请通过以下渠道**私下**报告：

- **邮箱**：security@stepwork.dev（占位，待正式域名启用）
- **备用**：通过 GitHub Security Advisories 的 "Report a vulnerability" 功能提交私有报告

## 报告应包含的信息

为帮助我们快速定位与修复，请在报告中尽量包含：

1. **漏洞类型**（如：权限绕过、路径穿越、注入、XXE、敏感信息泄露等）
2. **受影响的模块/路径**（如 `worker/runtime/rpc.py`、`apps/desktop/src-tauri/src/sidecar/spawn.rs`）
3. **复现步骤**（最小可复现 PoC，文本或代码）
4. **影响范围与危害评估**
5. **可能的修复建议**（可选）
6. **是否已公开披露**（是/否，以及披露渠道）

## 响应时限

| 阶段 | 时限 |
|---|---|
| 确认收到 | 72 小时内 |
| 初步评估（严重等级） | 7 个自然日内 |
| 修复或缓解方案 | 根据严重等级，高危 30 天内，中低危 90 天内 |
| 公开披露 | 修复发布后 7 天，或与报告者协商 |

## 严重等级参考

- **Critical**：远程未授权代码执行、完整数据泄露
- **High**：权限提升、凭据泄露、越权访问
- **Medium**：受限信息泄露、本地 DoS
- **Low**：需要复杂条件的安全弱点

## 安全更新

安全更新将以 GitHub Security Advisories + Release Notes 形式发布。关键漏洞会同步到 `SECURITY.md` 顶部的 "Known Advisories" 列表（当前为空）。

## 范围说明

以下内容**不在**漏洞奖励范围内，但仍欢迎报告：

- 第三方依赖已知漏洞（请直接向对应上游报告，我们会跟踪升级）
- 社会工程、物理访问攻击
- 需要 root/admin 权限的本地攻击
- 已列入 `docs/adr/` 的"显式不做"项（如 ADR-008 明确不做反检测）

感谢您对 STEPWORK 安全的贡献。
